## Elite Best Practices

### Type-First Python
- **Type hints everywhere**: `def process(data: list[dict[str, Any]]) -> pd.DataFrame:`
- **Pydantic models for all structured data**: Validation, serialization, documentation in one
- **`TypeAlias` for complex types**: `type UserData = dict[str, list[int]]`
- **`typing.Protocol`** for structural typing — duck typing with type safety
- **`@overload`** for functions with different return types based on input
- **Generic functions**: `def first[T](items: list[T]) -> T | None:`
- **`dataclasses` for internal data**: When you don't need validation (lighter than Pydantic)

### Data Science & ML Patterns
- **Reproducibility**: Set random seeds (`np.random.seed`, `torch.manual_seed`), pin library versions
- **Data pipeline**: raw data → cleaned → features → model input — each step a function
- **Train/test split early**: Before any data exploration to prevent leakage
- **Feature engineering in functions**: Named, documented, testable — not notebook cells
- **Model versioning**: Save with metadata — model file + config + metrics + training date
- **Evaluation before deployment**: Precision, recall, F1 — not just accuracy
- **Config-driven experiments**: Hyperparams in YAML/JSON config — not hardcoded

### Flask Best Practices
- **Application factory pattern**: `create_app()` function — configurable, testable
- **Blueprints for routes**: Group by feature — `auth_bp`, `api_bp`, `admin_bp`
- **Request validation**: Pydantic or Marshmallow schemas for all endpoints
- **Error handlers**: `@app.errorhandler(404)` with consistent JSON responses
- **Configuration classes**: `DevelopmentConfig`, `ProductionConfig`, `TestingConfig`
- **CORS**: `flask-cors` with explicit origins
- **Background tasks**: Celery for heavy processing — never block the request

### FastAPI Patterns (for your future / upgrades)
- **Auto-generated OpenAPI docs**: `/docs` endpoint for free
- **Dependency injection**: `Depends()` for auth, DB sessions, rate limiting
- **Pydantic models as request/response**: Type-safe API contracts
- **Async support**: `async def` routes for I/O-bound operations
- **Background tasks**: `BackgroundTasks` parameter for fire-and-forget operations

### Error Handling
- **Catch specific exceptions**: `except ValueError` not bare `except:`
- **Custom exceptions**: `class NotFoundError(Exception): ...` with context
- **`raise ... from ...`**: Chain exceptions — preserve the original traceback
- **Context managers**: `with` for file handles, DB connections, locks — always auto-cleanup
- **Logging over print**: `logging.getLogger(__name__)` — structured, leveled, configurable

### Testing
- **pytest**: Standard — not unittest
- **Fixtures**: `conftest.py` for shared setup — DB connections, test clients
- **Parametrize**: `@pytest.mark.parametrize` for testing multiple inputs
- **Coverage**: `pytest-cov` for coverage reports — aim for 80%+
- **Mocking**: `unittest.mock.patch` for external dependencies

### Data Processing
- **Pandas idioms**: Vectorized operations over loops — `df.apply()` over `for row in df.iterrows()`
- **Method chaining**: `df.query(...).groupby(...).agg(...)` — readable pipelines
- **NumPy for computation**: Use NumPy arrays for numerical operations — 100x faster than lists
- **Memory management**: `dtype` specification, chunked reading for large CSVs
- **Polars for speed**: Consider Polars for large datasets — faster than Pandas for pure data transforms

## Anti-Patterns (NEVER do these)

- Don't use mutable default arguments: `def f(x=[]):` → use `def f(x=None): x = x or []`
- Don't use `import *` — explicit imports always
- Don't use global variables for state — pass as parameters or use classes
- Don't suppress exceptions: `except: pass` hides bugs
- Don't use `%` string formatting — use f-strings
- Don't use `os.path` — use `pathlib.Path`
- Don't write untyped functions — type hints are documentation
- Don't use Jupyter notebooks for production code — extract to `.py` modules
- Don't mix tabs and spaces — use 4 spaces always (Black formatter handles this)
- Don't use `print()` for logging in production
