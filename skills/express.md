## Elite Best Practices

### Architecture Patterns
- **Layered architecture**: Route → Controller → Service → Repository/DB — each layer has single responsibility
- **Dependency injection**: Pass services into controllers — makes testing trivial
- **Feature-based folders**: Group by resource, not by type — `users/controller.ts`, `users/service.ts`, `users/routes.ts`
- **Thin route files**: Routes only define HTTP method + path + middleware chain → call controller
- **No Express in business logic**: Services should be framework-agnostic — Express stays in routes/controllers

### Middleware Mastery
- **Order matters**: Security → CORS → Body parsing → Auth → Rate limiting → Routes → Error handler
- **Composition**: `[auth, validate(schema), rateLimit]` — compose middleware arrays per route
- **Async middleware**: Use `express-async-errors` or wrap handlers — unhandled rejections crash the server
- **Request enrichment**: Auth middleware adds `req.user`, validation middleware adds `req.validated`
- **Short-circuit**: `return res.status(401).json(...)` — don't forget `return` to stop execution

### Request Validation
- **Validate at the boundary**: Every POST/PUT/PATCH body through Zod schema
- **Param validation**: `z.object({ id: z.string().uuid() })` for route params
- **Query validation**: `z.object({ page: z.coerce.number().positive().default(1) })`
- **Type-safe handlers**: Validated request → typed object — no `req.body.whatever` guessing
- **Sanitize output**: Don't return raw DB records — map to response DTOs

### Error Handling
- **Centralized error middleware**: Single `(err, req, res, next)` handler at the end
- **Custom error classes**: `BadRequestError`, `NotFoundError`, `UnauthorizedError` extending base `AppError`
- **Consistent format**: `{ error: { message: string, code: string, details?: unknown } }`
- **HTTP status codes**: Use correctly — 400 validation, 401 auth, 403 permission, 404 not found, 409 conflict, 422 unprocessable, 429 rate limit, 500 internal
- **Never expose stack traces in production**: `NODE_ENV === 'production'` → generic message
- **Log everything server-side**: Error + request context + user ID + timestamp

### Security Best Practices
- **`helmet()`**: Security headers in one line — CSP, HSTS, X-Frame-Options, etc.
- **CORS whitelist**: `cors({ origin: ['https://yourdomain.com'] })` — never `*` in production
- **Rate limiting tiers**: Stricter limits on auth endpoints (login, register, password reset)
- **Input sanitization**: Prevent NoSQL injection, XSS — validate and escape
- **API key auth**: For service-to-service — `x-api-key` header + middleware check
- **JWT**: Verify signature + expiration + issuer — never trust unvalidated tokens
- **HTTPS only**: Redirect HTTP → HTTPS in production

### Database Patterns
- **Connection pooling**: Reuse DB connections across requests — never connect per request
- **Transactions**: Wrap multi-step mutations in transactions — partial writes corrupt data
- **Prepared statements**: Parameterized queries always — never string interpolation with SQL
- **Migration files**: Version-controlled schema changes — never manual ALTER TABLE

### Performance
- **Compression**: `compression()` middleware for gzip/brotli — reduces response size 70-90%
- **Caching**: `Cache-Control` headers for GET responses — `stale-while-revalidate` pattern
- **Pagination**: Always paginate list endpoints — `?page=1&limit=20` with total count
- **Lean responses**: Return only needed fields — don't send entire DB rows
- **Keep-alive connections**: Enable for databases and external HTTP clients

### API Design
- **RESTful resource URLs**: `/api/v1/quotes`, `/api/v1/quotes/:id` — nouns not verbs
- **Versioning**: `/api/v1/` prefix — don't break existing clients
- **Consistent response envelope**: `{ data: T, meta: { page, total, limit } }` for lists
- **HATEOAS (optional)**: Include `_links` for discoverability in public APIs
- **Idempotent operations**: PUT/DELETE should be safe to retry

## Anti-Patterns (NEVER do these)

- Don't put business logic in route handlers — use service layer
- Don't use synchronous file operations in request handlers
- Don't trust `req.body` without validation
- Don't catch errors and silently continue
- Don't use `res.send()` after `res.json()` — response already sent
- Don't store sessions in memory — use Redis or database
- Don't return raw database errors to clients
- Don't use `*` CORS in production
- Don't forget to call `next()` in middleware (or send a response)
- Don't chain `.then()` in Express 5+ — use async/await
