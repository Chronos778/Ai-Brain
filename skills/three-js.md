## Elite Best Practices

### R3F Architecture
- **Declarative scene graph**: Components map to Three.js objects ‚Äî `<mesh>`, `<boxGeometry>`, `<meshStandardMaterial>`
- **Canvas setup**: `<Canvas>` auto-creates scene, camera, renderer ‚Äî configure via props
- **Camera control**: Use `<OrbitControls>` from Drei, or custom controls for specific UX (your TouchPanControls)
- **Scene composition**: Break 3D scenes into component files like regular React ‚Äî `<HeroSection>`, `<TimelineSection>`
- **Responsive canvas**: Track viewport with `useThree(state => state.size)` ‚Äî recalculate FOV/positions
- **Pointer events**: R3F supports `onClick`, `onPointerOver` on meshes natively ‚Äî use for interactivity

### Performance Optimization
- **Instancing**: `<instancedMesh>` for 100+ identical objects ‚Äî single draw call
- **Geometry merging**: `mergeBufferGeometries` for static geometry ‚Äî reduces draw calls dramatically
- **LOD (Level of Detail)**: `<Lod>` from Drei ‚Äî swap high/low poly models based on camera distance
- **Texture compression**: KTX2 format with `useKTX2` loader ‚Äî smaller files, GPU-native
- **Frustum culling**: Enabled by default ‚Äî objects outside camera view skip rendering
- **Object pooling**: Reuse mesh instances instead of creating/destroying ‚Äî especially for particles
- **Limit pixel ratio**: `<Canvas dpr={[1, 2]}>` ‚Äî cap at 2x even on 3x screens
- **Offscreen rendering**: Use `OffscreenCanvas` for web worker rendering in heavy scenes
- **Dispose everything**: `useEffect` cleanup must dispose geometries, materials, textures
- **Drei's `<Preload>`**: Preload all textures/models on first frame to avoid jank

### GSAP + Three.js Integration
- **Camera flythrough**: Animate camera position/rotation with `gsap.to(camera.position, { ... })`
- **Timeline sequences**: `gsap.timeline()` for section-by-section camera journeys
- **ScrollTrigger**: Pin 3D canvas and animate on scroll position ‚Äî immersive scrollytelling
- **Smooth easing**: `power2.inOut` for camera transitions, `elastic.out` for playful bounces
- **Kill animations on unmount**: `timeline.kill()` in useEffect cleanup to prevent memory leaks
- **Ref-based animation**: Animate `ref.current.position` not React state ‚Äî avoid re-renders

### Lighting & Materials
- **Environment maps**: `<Environment preset="city" />` from Drei for instant realistic lighting
- **PBR materials**: `<meshStandardMaterial>` or `<meshPhysicalMaterial>` for realistic surfaces
- **HDRI backgrounds**: Use `.hdr` environment maps for reflections and ambient lighting
- **Shadowmap**: Enable `shadows` on Canvas, add `castShadow`/`receiveShadow` to meshes
- **Contact shadows**: Drei's `<ContactShadows>` for cheap ground shadows

### Model Loading
- **GLTF/GLB**: Standard format ‚Äî use `useGLTF` from Drei with Suspense
- **Draco compression**: `useGLTF(url, '/draco/')` ‚Äî 90% smaller model files
- **Preload models**: `useGLTF.preload('/model.glb')` outside component ‚Äî start loading immediately
- **Transform on load**: Apply scale, position, rotation in the `useGLTF` callback or `<primitive>` props
- **GLTF ‚Üí JSX**: Use `npx gltfjsx model.glb` to generate declarative React components from models

### 3D Typography
- **Text3D**: Drei's `<Text3D>` with custom fonts (JSON format from Facetype.js)
- **Billboard text**: `<Text>` from Drei for always-facing-camera labels
- **HTML overlays**: `<Html>` from Drei for DOM elements anchored to 3D positions
- **Custom font meshes**: ExtrudeGeometry from SVG paths for fully custom 3D text

### Mobile Optimization
- **Reduce geometry complexity**: Lower subdivision on mobile ‚Äî check `useThree` viewport size
- **Simpler materials**: Use `<meshBasicMaterial>` on mobile ‚Äî skip PBR if not needed
- **Disable shadows on mobile**: `shadows={!isMobile}` on Canvas
- **Touch controls**: Custom touch handlers for pan/rotate/zoom ‚Äî default OrbitControls works but isn't optimal
- **Adaptive DPR**: `<Canvas dpr={[1, isMobile ? 1.5 : 2]}>` ‚Äî save GPU on mobile

## Anti-Patterns (NEVER do these)

- Don't create new objects in `useFrame` ‚Äî reuse with `useRef`/`useMemo`
- Don't forget to dispose resources on unmount (geometry, material, texture)
- Don't load uncompressed textures ‚Äî use KTX2 or at least WebP/AVIF textures
- Don't skip frustum culling for large scenes
- Don't animate React state for 3D ‚Äî animate refs directly via `useFrame` or GSAP
- Don't load models synchronously ‚Äî always `Suspense` + `useGLTF`
- Don't use raw Three.js imperatively when R3F provides a declarative equivalent
- Don't forget `<Preload all />` in Canvas for smooth initial render
- Don't put heavy computation in `useFrame` without throttling ‚Äî it runs 60fps

## Reference (from official docs)

### Core Concepts
- Scene graph: Scene ‚Üí Meshes (Geometry + Material) ‚Üí rendered by Camera via Renderer
- Always dispose of geometries, materials, and textures when done
- Use `requestAnimationFrame` for the render loop (or R3F handles it)
- BufferGeometry for custom geometry (more performant than legacy Geometry)

### With React (R3F)
- React Three Fiber makes Three.js declarative ‚Äî components map to Three objects
- `<Canvas>` sets up scene, camera, and renderer automatically
- Use Drei helpers: `OrbitControls`, `Environment`, `Text`, `Html`
- `useFrame` hook for per-frame updates (animation loop)
- `useLoader` / `useTexture` / `useGLTF` for asset loading
