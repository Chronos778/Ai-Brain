## Elite Best Practices

### Event Loop & Performance
- **Never block the event loop**: No `fs.readFileSync`, no CPU-heavy computation in request handlers
- **Use `worker_threads`** for CPU-intensive work (ML inference, image processing, heavy computation)
- **Streams everywhere**: `fs.createReadStream` → `pipe` → response for file serving
- **Backpressure handling**: Always handle `drain` events when writing streams
- **Connection pooling**: Reuse database connections — never create per-request connections
- **Clustering**: `cluster` module or PM2 to utilize all CPU cores in production

### Async Patterns
- **Always `await` Promise.all()** for independent async operations — never sequential `await`
- **`AbortController`** for cancellable operations (fetch timeout, DB query timeout)
- **Error handling in async**: Every `await` should be in try/catch or use `.catch()` on the promise
- **`unhandledRejection` handler**: Global safety net — log and exit gracefully
- **`AsyncLocalStorage`**: For request-scoped context (request ID, user session) without passing through every function

### File System
- **Always `path.join()` / `path.resolve()`** — never string concatenation for paths
- **`fs/promises`** for all file operations — the async versions
- **`import.meta.url`** for `__dirname` equivalent in ES modules: `new URL('.', import.meta.url).pathname`
- **`chokidar`** for file watching — `fs.watch` has platform-specific quirks
- **Temp files via `os.tmpdir()`** — never hardcode `/tmp`

### Security
- **Validate all input**: Zod for API payloads, environment variables
- **Never `eval()` or `new Function()`** — arbitrary code execution vulnerability
- **`crypto.randomUUID()`** for IDs — not `Math.random()`
- **Rate limiting**: Express middleware or API gateway level — protect against abuse
- **Secrets in env vars**: `.env.local` gitignored, production in Vercel/Firebase/platform env config
- **Dependency audit**: `npm audit --production` — only audit prod dependencies
- **No `child_process.exec`** with user input — use `execFile` or `spawn` with argument arrays

### Package Management
- **Lock files committed**: `package-lock.json` (npm) or `bun.lockb` (Bun) — always
- **Exact versions for critical deps**: `"react": "19.0.0"` not `"^19.0.0"` for core framework
- **Separate devDependencies**: Build tools, linters, test frameworks — not in production bundle
- **Engine constraints**: `"engines": { "node": ">=18" }` in package.json
- **Audit regularly**: `npm audit` before deployments

### Error Handling
- **Custom error classes**: `class NotFoundError extends Error { statusCode = 404 }`
- **Structured error logging**: JSON logs with error type, stack, request context
- **Graceful shutdown**: Handle `SIGTERM` — close connections, flush logs, then `process.exit(0)`
- **Health checks**: `/health` endpoint that verifies DB connection, external services
- **Never expose stack traces in production** — generic user message, detailed server logs

### Environment & Configuration
- **`process.env.NODE_ENV`** for environment detection — 'development', 'production', 'test'
- **Config validation at startup**: Parse all env vars through Zod schema — fail fast with clear errors
- **`.env.example`** committed — template for required env vars without values
- **Never default secrets**: `process.env.SECRET ?? 'fallback'` — if no secret, crash loudly

## Anti-Patterns (NEVER do these)

- Don't block the event loop with synchronous operations
- Don't use `eval()` or `Function()` constructor
- Don't store secrets in code — use environment variables
- Don't use `var` — use `const`/`let`
- Don't ignore errors in callbacks or promises
- Don't use `require()` in ES module projects — use `import`
- Don't use `console.log` in production — use a structured logger (pino, winston)
- Don't install packages globally — use `npx`/`bunx` or project-level installs
- Don't trust `req.body` without validation
- Don't ignore `unhandledRejection` events
