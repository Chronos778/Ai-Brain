## Elite Best Practices (Vercel Web Interface Guidelines)

### Design System Tokens
- **Consistent spacing**: Use Tailwind's scale (4, 8, 12, 16, 20, 24...) — never arbitrary `[13px]`
- **Custom color palette**: Define in `tailwind.config` using CSS variables for light/dark switching
- **Typography scale**: Use Tailwind's built-in sizes — `text-sm`, `text-base`, `text-lg` — customize only if needed
- **Border radius scale**: Keep consistent — pick 2-3 radius values and stick with them
- **Shadows**: Define custom shadow tokens — `shadow-card`, `shadow-dropdown` — not inline arbitrary values

### Responsive Design
- **Mobile-first always**: Base styles are mobile, progressive enhancement with breakpoints
- **Container queries**: Use `@container` for component-level responsiveness (not just viewport)
- **Fluid typography**: `text-base md:text-lg lg:text-xl` for graceful scaling
- **Hide/show**: `hidden md:block` not `display: none` media queries
- **Touch targets**: Minimum `h-10 w-10` (40px) for all interactive elements on mobile

### Dark Mode & Theming
- **CSS variables for colors**: `--primary: 220 90% 56%` in `:root` and `.dark`
- **`color-scheme: dark`** on `<html>` for native dark scrollbars, inputs, selects
- **`<meta name="theme-color">`** matching page background — updates browser chrome
- **Consistent contrast**: WCAG AA minimum (4.5:1 for text, 3:1 for large text)
- **Never hardcode colors** in component classes — always use theme tokens

### Component Patterns with CVA
```
const button = cva('inline-flex items-center justify-center rounded-md font-medium transition-colors', {
  variants: {
    variant: { default: 'bg-primary text-primary-foreground', outline: 'border border-input bg-transparent' },
    size: { sm: 'h-8 px-3 text-xs', md: 'h-10 px-4 text-sm', lg: 'h-12 px-6 text-base' }
  },
  defaultVariants: { variant: 'default', size: 'md' }
})
```

### Animation & Motion
- **Honor `prefers-reduced-motion`**: `motion-safe:animate-fade-in` or `motion-reduce:animate-none`
- **Compositor-friendly only**: Animate `transform` and `opacity` — never `width`, `height`, `top`, `left`
- **Never `transition-all`**: List properties explicitly — `transition-colors`, `transition-transform`
- **Set `transform-origin`**: Correct origin for scale/rotate animations
- **Interruptible animations**: Transitions should respond to user input mid-animation

### Accessibility
- **Focus states**: `focus-visible:ring-2 focus-visible:ring-ring` — never `outline-none` without replacement
- **Interactive hover**: `hover:bg-accent` — every clickable element needs visual feedback
- **Disabled states**: `disabled:opacity-50 disabled:pointer-events-none`
- **`sr-only` for screenreader text**: Icon-only buttons need hidden labels
- **Color is not the only indicator**: Don't convey meaning through color alone

### Layout Patterns
- **Flex/Grid over JS**: Layout with Tailwind utilities — never JavaScript measurement for layout
- **`min-w-0` on flex children**: Required for text truncation inside flex containers
- **Safe areas**: `env(safe-area-inset-*)` for notch-aware layouts
- **Overflow prevention**: `overflow-x-hidden` on scroll containers, not the body
- **Gap over margin**: `gap-4` on flex/grid parents instead of `mb-4` on children

### Typography (Vercel Guidelines)
- **Proper ellipsis**: Use `…` not `...` — `truncate` class handles this in UI
- **Curly quotes**: `"text"` not `"text"` — `text-pretty` or `text-balance` on headings
- **Tabular numbers**: `font-variant-numeric: tabular-nums` for number columns
- **`text-wrap: balance`**: On headings to prevent widows
- **Loading states**: End with `…` — "Loading…", "Saving…"

### Content Handling
- **Truncation**: `truncate` (single line), `line-clamp-3` (multi-line)
- **Empty states**: Always handle — never render broken UI for empty arrays
- **Long content**: `break-words` or `break-all` for user-generated content
- **Images**: Always `width` + `height` attributes — prevent CLS

## Anti-Patterns (NEVER do these)

- Don't use `@apply` to recreate component classes — use actual component extraction
- Don't fight Tailwind with custom CSS — work with the utility system
- Don't override Tailwind's reset/preflight
- Don't use arbitrary values `[...]` when a design token exists
- Don't use `transition-all` — list specific properties
- Don't use `outline-none` without a focus-visible replacement
- Don't use `style={{ }}` for things Tailwind can handle
- Don't nest Tailwind classes in CSS files with `@apply` chains — that's just CSS with extra steps
- Don't use `!important` via `!` prefix unless absolutely forced by third-party CSS
- Don't hardcode color values in classes — always use theme tokens

## Reference (from official docs)

### Core Principles
- Utility-first: compose designs directly in markup
- Don't abstract too early — duplicate utility classes are fine initially
- Extract components (React/Vue/etc.) before extracting CSS classes
- Use `@apply` sparingly — it defeats the purpose of utility-first

### Customization
- Extend (don't override) the default theme in `tailwind.config`
- Use CSS variables for dynamic theming
- Use `@layer` for proper specificity ordering
- Define custom colors as CSS variables for light/dark mode switching
